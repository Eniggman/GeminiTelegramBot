## 💭 Обработка сообщений

### Главный обработчик: `handle_message()`

**Порядок обработки:**

```
Сообщение → Проверка доступа → Определение типа → Обработка
```

### Специальные префиксы

| Префикс | Действие | Режим ожидания |
|---------|----------|----------------|
| `.` | Сброс контекста | Нет |
| `Пр`, `Перевод` | Перевод текста | Да — ждёт текст |
| `К`, `Картинка` | Генерация изображения | Да — ждёт описание |
| `Р` | Редактирование фото | Нет (подпись к фото) |

### Режимы ожидания (user_modes)

**Режим перевода (`translate`):**
1. Отправляешь `Пр` → бот ждёт текст
2. Отправляешь текст → бот переводит
3. Режим автоматически сбрасывается

**Режим генерации (`image_gen`):**
1. Отправляешь `К` → бот ждёт описание
2. Отправляешь описание → бот генерирует картинку
3. Режим автоматически сбрасывается

**Режим анализа фото:**
1. Отправляешь фото (без подписи) — появляются кнопки
2. Нажимаешь `🔍 Анализировать` — бот описывает фото
3. Или отправляешь фото с вопросом (в подписи или реплаем) — бот отвечает

### Обычный чат
1. Получаем/создаём сессию для пользователя
2. Отправляем сообщение в Gemini с retry-логикой
3. Обрабатываем Markdown для Telegram
4. Разбиваем длинные ответы на части

## 🖼️ Генерация изображений

### Модели
```python
IMAGE_MODELS = {
    'pro': 'gemini-3-pro-image-preview',
    'flash': 'gemini-2.5-flash-image'
}
```

### Функции

#### `generate_image(prompt, user_id)`
- Генерирует изображение по текстовому описанию
- Использует `genai_image` (google.genai)
- Возвращает `bytes` изображения или `None`

#### `edit_image(image_bytes, prompt, user_id)`
- Редактирует существующее изображение по промпту
- Принимает bytes исходного изображения
- Возвращает `bytes` нового изображения или `None`

### Обработчик фото: `handle_photo()`

| Подпись | Действие |
|---------|----------|
| `Р` + текст | Редактирование изображения |
| Без подписи | Анализ изображения (с кнопками) |

## 🎤 Голосовые сообщения

### Обработчик: `handle_voice()`

**Алгоритм:**
1. Скачивается аудиофайл (OGG)
2. Загружается в Gemini как audio
3. Gemini транскрибирует и отвечает

## 🚀 Запуск бота

### Точка входа
```python
if __name__ == '__main__':
    load_activity_log()     # Загрузка логов активности
    load_users()            # Загрузка пользователей
    
    # Регистрация обработчиков
    application.add_handler(CommandHandler("start", start))
    application.add_handler(CommandHandler("help", help_command))
    # ... остальные команды
    
    application.add_handler(MessageHandler(filters.VOICE, handle_voice))
    application.add_handler(MessageHandler(filters.PHOTO, handle_photo))
    application.add_handler(MessageHandler(filters.TEXT & (~filters.COMMAND), handle_message))
    
    application.run_polling(drop_pending_updates=True)
```

### post_init (Меню команд)
При запуске устанавливается меню команд бота через `set_my_commands()`.

## 📝 Заметки для разработки

### Частые проблемы

1. **Бот не отвечает**
   - Проверь `check_access()` — пользователь может быть не в списке
   - Проверь логи на ошибки API

2. **Ошибка 403 Permission**
   - Модель может быть недоступна — измени на стабильную версию
   - Проверь API ключ

3. **Длинные ответы обрезаются**
   - Telegram лимит 4096 символов
   - `split_message()` должна разбивать текст

### Добавление новой команды
1. Создать async функцию-обработчик
2. Добавить `CommandHandler` в секцию запуска
3. Обновить `post_init` для меню команд
4. Обновить `/help`
